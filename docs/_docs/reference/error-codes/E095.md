# E095: Expected Type Bound Or Equals

This error is emitted when a type parameter or type member definition has invalid syntax - expecting `=`, `>:`, or `<:` but finding something else.

Type parameters and abstract types may be constrained by type bounds:
- `=` for type aliases
- `<:` for upper type bounds (subtype constraint)
- `>:` for lower type bounds (supertype constraint)

---

## Example

```scala sc:fail
type MyType : Int
```

### Error

```scala sc:nocompile
-- [E095] Syntax Error: example.scala:1:12 
1 |type MyType : Int
  |            ^
  |            '=', '>:', or '<:' expected, but ':' found
```

### Solution 

```scala sc:compile 
// Use = for type alias
type MyType = Int
``` 

```scala sc:compile 
// Use <: for upper bound
type MyType <: AnyVal
``` 

```scala sc:compile 
// Use >: for lower bound
type MyType >: Nothing
``` 

```scala sc:compile 
// Combine bounds
type MyType >: Nothing <: AnyVal
```
