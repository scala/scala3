# E233: Increasing Match Type Reduction

This error is emitted when reducing a match type leads to a reduction step whose
arguments keep growing without making progress towards a base case, so the
reduction would not terminate.

The divergence check compares the arguments of successive reduction steps. If the
*covering set* (the set of type variables and constants occurring in the
arguments) stays the same while the structural sizes of the arguments are
*dominated* by the next step — every argument is at least as large and at least
one is strictly larger — the reduction is considered divergent and reported
instead of looping.

---

## Example

```scala sc:fail
type Wrap[X] = X match
  case AnyVal => Wrap[List[X]]
  case AnyRef => Wrap[List[X]]

type Test = Wrap[Int]
```

Here `Wrap[Int]` reduces to `Wrap[List[Int]]`, then `Wrap[List[List[Int]]]`, and
so on: the argument grows on every step while the covering set stays the same, so
the reduction never reaches a result.

### Error

```scala sc:nocompile
-- [E233] Type Error: example.scala -------------------------------------------
 |  Match type reduction failed due to potentially infinite recursion in the reduction steps.
```

### Solution

Make sure every recursive case eventually reaches a non-recursive (base) case,
so that the arguments shrink towards termination rather than growing:

```scala sc:compile
type Unwrap[X] = X match
  case List[t] => Unwrap[t]
  case _       => X

type Test = Unwrap[List[List[Int]]] // = Int
```
