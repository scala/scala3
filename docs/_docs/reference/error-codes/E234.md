# E234: Cyclic Match Type Reduction

This error is emitted when reducing a match type revisits a set of arguments that
has already been seen earlier in the same reduction chain. Such a cycle means the
reduction would loop forever, so it is reported instead.

Unlike [E233](E233.md), where the arguments keep growing, here the reduction
returns to a previously encountered state of the same size, forming a cycle.

---

## Example

```scala sc:fail
type Loop[X] = X match
  case Int    => Loop[Float]
  case Float  => Loop[Double]
  case Double => Loop[String]
  case String => Loop[Int]
  case _      => String

type Test = Loop[Int]
```

`Loop[Int]` reduces to `Loop[Float]`, `Loop[Double]`, `Loop[String]`, and back to
`Loop[Int]`, cycling forever without reaching the `case _` base case.

### Error

```scala sc:nocompile
-- [E234] Type Error: example.scala -------------------------------------------
 |  Match type reduction failed due to a cycle in the reduction steps.
```

### Solution

Restructure the recursion so that each step makes progress towards a base case
instead of cycling back to an earlier state:

```scala sc:compile
type Widen[X] = X match
  case Int    => Long
  case Long   => Double
  case Double => Double

type Test = Widen[Int] // = Long
```
