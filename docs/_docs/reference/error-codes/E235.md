# E235: Match Type Reduction With Error Arguments

This error is emitted when a match type cannot be reduced because one or more of
its arguments have themselves reduced to error types. The match type reduction
cannot proceed meaningfully until the underlying errors in the arguments are
fixed, so it is reported rather than producing further spurious errors.

This typically appears together with another, root-cause error coming from the
argument that failed to reduce.

---

## Example

```scala sc:fail
type F[A] = A match
  case Int => F[F[A]]
  case _   => F[F[A]]

type Test = F[F[Int]]
```

The inner application fails to reduce, so the outer reduction step is asked to
proceed with an argument that is already an error type.

### Error

```scala sc:nocompile
-- [E235] Type Error: example.scala -------------------------------------------
 |  Match type reduction failed due to error in arguments.
```

### Solution

Fix the errors reported for the arguments first; once the arguments reduce
successfully, the enclosing match type can be reduced as well:

```scala sc:compile
type Elem[X] = X match
  case List[t] => t
  case _       => X

type Test = Elem[List[Int]] // = Int
```
