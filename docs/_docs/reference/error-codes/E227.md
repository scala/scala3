# E227: Private Shadows Field

This warning is emitted when a private field shadows an inherited field with the same name.

Shadowing inherited fields can make code confusing because it hides which member is being used.
Renaming the private field usually makes intent explicit and avoids subtle mistakes.

This warning is enabled with the `-Wshadow:private-shadow` compiler flag.

---

## Example

```scala sc:fail sc-opts:-Wshadow:private-shadow,-Werror,-explain
class Base(var x: Int)

class Derived(x: Int) extends Base(x)
```

### Warning

```scala sc:nocompile
-- [E227] Naming Warning: example.scala:3:14 -----------------------------------
3 |class Derived(x: Int) extends Base(x)
  |              ^
  |        value x in class Derived shadows field x inherited from class Base
  |-----------------------------------------------------------------------------
  | Explanation (enabled by `-explain`)
  |- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -
  | A private field shadows an inherited field with the same name.
  | This can lead to confusion as the inherited field becomes inaccessible.
  | Consider renaming the private field to avoid the shadowing.
   -----------------------------------------------------------------------------
```

### Solution

Rename the constructor parameter so it no longer shadows the inherited field:

```scala sc:compile sc-opts:-Wshadow:private-shadow,-Werror
class Base(var x: Int)

class Derived(x0: Int) extends Base(x0)
```
